## Summary

FolderSize v0.2.4 fixes stale folder sizes after directory contents change and corrects the release documentation/packaging from the earlier v0.2.2/v0.2.3 attempts.

## Important Changes

- Fixed visible Explorer rows staying on old folder-size values after files were added or changed.
- Added a throttled background refresh path so visible cached folders self-correct quickly even if a shell notification is missed.
- Tightened direct scanner completion behavior so incomplete, cancelled, or errored scans are not cached as final sizes.
- Kept Everything optional: final displayed sizes come from direct filesystem scans, not Everything IPC.
- Removed the stale README warning that incorrectly said not to use the project.
- Fixed release ZIP install/uninstall scripts so they work when extracted next to `foldersize.dll`.
- Added SHA-256 checksums and build provenance (`BUILD-INFO.txt`) to release artifacts.

## Validation

- GitHub Actions builds the DLL on `windows-latest` and runs the CTest suite before publishing.
- Local Explorer UI validation confirmed a folder row updated from `31 KB` to `10.0 MB` after adding a 10 MB file.
- The release ZIP includes `foldersize.dll`, `foldersize.pdb`, install/status/uninstall scripts, docs, README, LICENSE, and build metadata.

## Install

Download `foldersize-v0.2.4-win64.zip`, extract it, and run `install.bat` as Administrator. Use `status.bat` to confirm COM registration and Explorer loading.
