## Summary

FolderSize v0.2.5 corrects README inaccuracies in the release artifact after the v0.2.4 runtime fix.

## Important Changes

- Clarified that the bundled build script expects Visual Studio 18 / 2026 Enterprise or Build Tools, even though the project itself uses MSVC, CMake, and vcpkg.
- Clarified that `install.bat` and `uninstall.bat` require Administrator privileges; `status.bat` is a reporting script.
- Corrected the documented DLL footprint from the stale `144 KB` value to approximately `170 KB`.
- Clarified manual uninstall instructions for extracted release ZIPs.
- Softened comparison-table wording to avoid overstating implementation details of the Windhawk mod.
- Clarified that the runtime environment is Windows Explorer plus the VC++ runtime, not Windows Explorer alone.

## Validation

- GitHub Actions builds the DLL on `windows-latest` and runs the CTest suite before publishing.
- The release ZIP includes the corrected README, `foldersize.dll`, `foldersize.pdb`, install/status/uninstall scripts, docs, LICENSE, and build metadata.

## Install

Download `foldersize-v0.2.5-win64.zip`, extract it, and run `install.bat` as Administrator. Use `status.bat` to confirm COM registration and Explorer loading.
